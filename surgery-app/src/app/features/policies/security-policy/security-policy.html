Política de Seguridad

Políticas de seguridad del sistema

Última actualización: {{ lastUpdated }} · SurgicalOS v1.0

1. Alcance y propósito

Esta política aplica a todos los usuarios del sistema SurgicalOS, incluyendo administradores, cirujanos, anestesiólogos y pacientes. El propósito es garantizar la confidencialidad, integridad y disponibilidad de la información clínica gestionada por el sistema.

2. Principios de seguridad

Mínimo privilegio
Cada rol de usuario accede únicamente a las funciones y datos estrictamente necesarios para su labor.
Seguridad por capas
Controles de seguridad aplicados en múltiples niveles: frontend (guards), backend (middleware) y base de datos (permisos).
Fallos seguros
Ante cualquier error, el sistema deniega el acceso por defecto y no expone datos sensibles.

3. Roles y control de acceso

Rol Acceso permitido
Administrador Acceso completo a todas las secciones del sistema
Cirujano Cirugías, calendario, pacientes, documentos
Anestesiólogo Cirugías, calendario, documentos
Paciente Calendario (solo lectura), documentos propios

4. Política de contraseñas

  • Longitud mínima de 8 caracteres.
  • Debe incluir al menos una mayúscula, una minúscula y un número.
  • Caducidad cada 90 días. El sistema solicitará cambio de contraseña.
  • No se permite reutilizar las últimas 5 contraseñas.
  • Bloqueo de cuenta tras 5 intentos fallidos consecutivos.

5. Gestión de sesiones

  • Las sesiones expiran automáticamente tras 60 minutos de inactividad.
  • Al cerrar el navegador se cierra la sesión activa.
  • No se permite el uso simultáneo de una misma cuenta desde múltiples dispositivos.
  • Toda sesión queda registrada con fecha, hora e IP de acceso.

6. Manejo de datos sensibles

La información clínica de los pacientes se clasifica como datos sensibles y se gestiona bajo los siguientes principios:

  • Almacenamiento cifrado en la base de datos (AES-256).
  • Transmisión exclusivamente sobre HTTPS/TLS.
  • Acceso restringido según el rol del usuario (mínimo privilegio).
  • Los documentos PDF se almacenan en servidores seguros con acceso autenticado.
  • No se comparte información con terceros sin consentimiento explícito del paciente.

7. Uso aceptable del sistema

  • El sistema es de uso exclusivo institucional para la gestión de cirugías.
  • Queda prohibido el acceso no autorizado a cuentas de otros usuarios.
  • No se permite subir documentos con contenido malicioso o no relacionado al ámbito clínico.
  • Los usuarios son responsables de mantener la confidencialidad de sus credenciales.
  • Cualquier incidente de seguridad debe reportarse al administrador del sistema.

8. Referencia normativa

Esta política se basa en los lineamientos del OWASP Top 10, las guías de implementación segura de OWASP, y los estándares de seguridad de la información del curso CE-1115 del Instituto Tecnológico de Costa Rica.
